通过 Github 的操作来说明工作区、暂存区、版本库之间的关系

1. git工作流三者关系

工作目录,暂存区(Index),版本库(HEAD)三者关系如图:

2.在git命令行下的操作

  1. 创建一个文件夹后,在文件夹中git init,此目录为工作区.或直接git clone 一个项目,项目的目录为工作区
  2. 在文件修改变动后git add即把变动文件转到到暂存区
  3. 最后git commit后即提交到了版本库

3. 在git hub上的操作

3.1添加到工作区

  1. 点左上角上+后如图所示,增加相应git工作区.
    Add 为添加本地已经存在的文件夹,选中相应的文件夹后,Add repository即可

    Create 为本地的文件夹创建新的git工作区.

    Clone主要是将自己github上的项目下载下来.

    3.2添加到暂存区和版本库

  2. 新建git本地工作区后,到相应的文件夹下修改文件.如创建一个readme.txt的文件.github客户端自动会识别已经修改的内容,并自己勾选增加到暂存区(git add),如不加入可以不勾选.下面commit即为git commit的操作.

    3.3github客户端将两个操作合并成一个界面.

    在提交前前后用git for windows的git bash查看状态git status如下图:
    提交前提示需要git add,提交后直接就完成提交.相当一次完成两个操作.
